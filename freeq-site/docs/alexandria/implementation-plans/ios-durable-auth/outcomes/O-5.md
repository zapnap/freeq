## Validation Criteria

- `fetchBrokerSession()` emits a `tracing` (Rust) / `print` or `os.Logger`
  (Swift) entry whenever credentials are auto-cleared, with the trigger reason
  (consecutive 401 count, last status code, time since last login).
- Integration test covers: WS connect happy path, WS connect timeout, TCP
  fallback after WS failure, idle-ping observed, dead-detection triggers
  reconnect after 90 s of silence.
- `freeq-sdk` tests can run against a local mock WebSocket server (no
  external dependency on `irc.freeq.at`).

## Motivation

We've tried to fix this several times. Without observability the next debug
session will be just as hard as this one. Tests pin the behaviour so we
don't regress when the SDK changes.

Could-tier because the Must outcomes deliver the user-visible win on their
own; this outcome is the durability of the fix.
