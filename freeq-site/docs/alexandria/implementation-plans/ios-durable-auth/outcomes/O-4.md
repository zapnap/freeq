## Validation Criteria

- WebSocket transport sends a PING (IRC- or WS-frame) every 45 s of idle, matching
  `freeq-sdk-js/src/transport.ts` (`PING_INTERVAL = 45000`).
- Connection is force-reconnected if no data is received for 90 s
  (`DEAD_TIMEOUT = 90000`).
- Reconnect uses exponential backoff `min(1000 * 2^n, 30000)` ms, identical
  to the JS transport.
- A cellular handoff (Wi-Fi → LTE → Wi-Fi) recovers the IRC session via
  transport-level reconnect without going back through the broker.

## Motivation

The JS Transport class is proven in production. Mirroring its parameters in
the Rust SDK gives us cellular-handoff resilience at the transport layer
without the iOS app having to drop and re-establish the entire `FreeqClient`
(which currently triggers a broker round-trip and a fresh SASL handshake).

This is a Should rather than a Must because O-1 + O-2 alone make the
manual-OAuth scenario rare; this outcome makes the experience smooth even
on flaky networks.
